Scopes
Eksempel på scopes:

- Se et scope som en stack

- Hver gang man hopper ind I et nyt scope med curly braces, hvad end det er et object, en function osv. creater den en ny stack, det kaldes for executional call stack som høre til præcis dette scope.
○ Den stack bliver så populated med alle variablerne, og så når vi leaver det scope "poppes" alle variablerne og scopet vil forsvinde


Strict mode
- skrives: "use strict"
- Strict mode forhindre at man kan gøre nogle bestemte ting man ellers normalt kan gøre
- I js er der et mode der hedder strict mode. Det skrives øverst af scope: "use strict";
- Det er et mode man kan bruge I sin kode som giver en fordel.
- Skrives det øverst vil de functions der skrives efterfølgende arve strict mode

Type coercion
- I Javascript er der noget der hedder type coercion
- Den prøver at ændre typerne så den KAN arbejde med dem (eksemplet I variabler med)
Fx:


I "2" + 2 giver den '22'. Her tolker den det altså som en string og giver os en string
I "4" / 2. Her tolker den det som tal og giver os altså et nummer
I javascript gælder også at:

- Altså 1 equals true
- True er bigger than zero er true
- Og false er less than 10 equals true
- Så type coercion er altså lidt skørt og vil man undgå denne behavior så skriv === I stedet for ==
- For man vil ikke have denne behavior hvor den evaluater true == 1 hvis ikke de er samme typer
- Så fremover brug altid 3 x === når man laver equality check for at undgå type coercion
- Type coercion converter altså typen så den kan compare dem.
- Så fremover lav tjekket før man vil converte det. Det er jo det type coercion gør. Så laver vi et hard equality check (===) på linjen før, så viser man at man ved hvad datatyper man arbejder med - Vil man tjekke omvendt (altså !=) skriver man:
- True !== 0 // hard not check, compare både value og types med === og !== - True != 1 // type coercion

- Hush, altså altid at compare både value og types med === og !==
Hard equality check kaldes også for strict equal
