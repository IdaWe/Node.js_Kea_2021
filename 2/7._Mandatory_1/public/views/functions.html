Vi arbejder med 4 slags functions når vi arbejder med Node.js:
- Normal function (function declaration)
- Anonymous function
- Arrow function
- Callback functions

Callback functions:
- Callbacks er functions som man passer som et argument, og hvis man ville kunne man så kalde det tilbage senere
- Forestil dig et tlf selskab: du ringer og der er optaget. Du kan enten vente på linjen eller de har måske et system (callback) hvor de ringer tilbage når det er din tur - og det er jo smart!
- Det smarte ved dette system er at man kan lave andre ting imens (for selve ventetiden er jo den samme)
- Det samme gælder for et program - det kan lave andre ting imens
- A callback er en function som vi passer som et argument i en anden function (til en anden?)
○ "the callback functionality" er noget vi muligvis gør, men et callback er bare selve det at vi passer en function som et argument
- Og vi kan passe functions som arguments I javascript fordi functions er 1. class citizens (- hvilket vil sige at functions bliver behandlet som en hvilken som helst anden value og derfor kan passes som argument)
- Callbacks er vigtige når man arbejder med node
- Modst fx I java hvor methods er 2. class citizens
- I node er der ikke nogen 2. class citizens(ifølge anders)


Hoisting / to hoist:
to hoist = to lift op
- Dette er en feature I javascript som "løfter" en funktionen op
- Når man deklerere en function bliver den automatisk hoistet
- Variable bliver hoistet hvis de er I et scope, men ikke hvis de befinder sig i selve filen
- Hele filen bliver gennemgået og løfter alle function declarations op til memoryspace I starten, så de kan kaldes fra alle steder I filen
