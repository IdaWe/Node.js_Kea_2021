REST API
- Rest api's pair HTTP methods med endpoints
- Og det smarte ved dette er, følger man en bestemt convention kan man faktisk se hvad det endpoint gør/burde gøre - også selvom man ikek selv har skrevet det, altsi API'en eller kan se koden, eller der er noget deokumentation. Så bare ved at se på to values; endpoint og http methods - skal man kunne forudsige hvordan det skal se ud, hvad det gør og hvad det giver dig.

- GET: retrieves ressources

- POST: creates a new ressource
○ Der er dog undtagelser hvor man ikke creater en new ressource, men vil bruge POST
○ Dette er tilfældet når vi gerne vil sende sensitive information, fx passwords
○ Rent teoretisk skulel det være en GET. MEN hvis vi brugte GET det blive sendt I URL'en og derfor være plain text. Det vil vi ikke. Så derfor bruger vi POST så det sendes I body
○ Det samme gælder hvis data er for stor til URL - for der er et limit

- DELETE: deletes/removes a ressource

- PUT/PATCH: updates a ressource
○ Ofte bliver PUT brugt når hele ressource skal updates
○ Ofte bliver PATCH brugt når det kun er en del af en ressource

- Når man snakker API og man siger collection kan det fx være users table - users er en collection
