Number vs. String

const:
- Const = en konstant
- Man kan ikke ændre denne variablen, den er altid fast
Const er immutable (det er en computer science term for at den ikke kan ændres
- Tænk over om en variabel skal ændres senere hen. Skal den ikke så lav den som const




let:
- Vi skal bruge let I stedet for var siger Anders
- Det hander om noget med deres scope
- let er nyere end var
Let kan ændres
Er mutable (det er en computer science term for at den kan ændres)
Mutable = noget man kan edit


Var vs. const/let
- Var scopede ikke særlig godt
Pga var var dårlig og det måske ville breake nogle apps hvis man ændrede den så lavede man const og let
