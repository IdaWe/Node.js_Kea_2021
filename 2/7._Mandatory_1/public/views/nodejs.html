Nodemon
- Det er irriterende at man skal stoppe go starte ens server / app selv hver gang man ændrer noget
- Npm Nodemon er et tool der kan dette
- "Nodemon is a tool that helps develop node.js based applications by automatically restarting the node application when file changes in the directory are detected."
- Kan installeres på 2 måder: globalt eller lokalt I ens projekt
○ Globally: it's outside of this project, it's something that we can use everywhere on our machine. Vi har first server og kan bruge det der, vi kan så også lave anden server og også bruge det der
§ Der er en node modules folder "I skyen" hvor nodemon vil 'eksistere/leve'. Så der er node modules I hvert project og der er så også en global node modules og det er her den globale installalation af nodemon vil "eksistere/leve"
○ Locally: it's then related to this unique project
- Der er for og imod begge måder
- Men ifølge anders giver det mening at holde kode sepereret fra build tools som fx nodemon; altså installer det globalt
- For at installere: ifølge siden(npm nodemon): npm install -g nodemon
- Installere ifølge anders: npm install nodemon -g
- Nodemon er til udvikling - det har ikke noget at gøre med production
○ Og KUN udvikling
○ Det er om hvordna man køre sin server (app), ændre filer osv
- Man kan tjekke nodemon er installere ved:
○ Skriv nodem I terminal og tryk på tab. For windows users færdiggør den så og skriver nodemon.cmd
○ For at tjekke ekstra: skriv nodemon.cmd og tryk enter og der skulle gerne komme en hjælpemenu frem
○ I stedet for at skrive det så skriv nodem og tryk tab så autocompleter den og skriver det der mangler ( jeg vidste ik der skulel stå cmd fx)
- For at køre serverne med nodemon skriv: nodemon.cmd app.js
- Minder en smule om REPL
